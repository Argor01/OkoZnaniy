# Добавление поля "Город проживания" для партнеров

## Что добавлено

### Бэкенд (Django)

1. **Модель User** (`apps/users/models.py`):
   ```python
   city = models.CharField(max_length=100, blank=True, null=True, verbose_name="Город проживания")
   ```

2. **UserSerializer** (`apps/users/serializers.py`):
   - Добавлено поле `city` в список полей сериализатора

3. **DirectorPersonnelViewSet** (`apps/director/views.py`):
   - Обновлен action `register` для обработки поля `city`
   - Добавлена валидация: город обязателен для партнеров
   - Поле `city` передается при создании пользователя

### Фронтенд (React)

1. **Интерфейс RegisterEmployeeRequest** (`frontend-react/src/features/director/api/types.ts`):
   ```typescript
   city?: string; // Город проживания для партнеров
   ```

2. **Компонент EmployeeRegistration** (`frontend-react/src/features/director/components/PersonnelManagement/EmployeeRegistration.tsx`):
   - Добавлено состояние `selectedRole` для отслеживания выбранной роли
   - Добавлено условное поле города, которое показывается только для партнеров
   - Добавлена валидация поля города (обязательное, минимум 2 символа)
   - Поле города передается в API только для партнеров

## Функциональность

### Условия отображения
- Поле "Город проживания" появляется **только** при выборе роли "Партнёр"
- Для других ролей (Администратор, Эксперт) поле не отображается

### Валидация
- **Фронтенд**: 
  - Обязательное поле для партнеров
  - Минимум 2 символа
- **Бэкенд**: 
  - Проверка обязательности города для роли 'partner'
  - Возврат ошибки если город не указан для партнера

### API
- **Endpoint**: `POST /api/director/personnel/register/`
- **Новое поле**: `city` (строка, опциональная)
- **Логика**: поле `city` сохраняется только если роль = 'partner'

## Пример использования

### Регистрация партнера
```json
{
  "email": "partner@example.com",
  "first_name": "Иван",
  "last_name": "Петров",
  "phone": "+7999123456",
  "role": "partner",
  "city": "Москва"
}
```

### Регистрация других ролей
```json
{
  "email": "admin@example.com",
  "first_name": "Анна",
  "last_name": "Сидорова",
  "role": "admin"
  // city не требуется и не передается
}
```

## Интерфейс

### Форма регистрации
1. Пользователь выбирает роль из выпадающего списка
2. При выборе "Партнёр" появляется дополнительное поле "Город проживания"
3. Поле имеет иконку местоположения и плейсхолдер "Москва"
4. При отправке формы поле валидируется

### Валидация в реальном времени
- Поле становится обязательным только для партнеров
- Показывается ошибка если поле пустое или содержит менее 2 символов
- Бэкенд возвращает ошибку если город не указан для партнера

## Тестирование

### Сценарии для тестирования

1. **Регистрация партнера с городом**:
   - Выбрать роль "Партнёр"
   - Заполнить все поля включая город
   - Проверить успешную регистрацию

2. **Регистрация партнера без города**:
   - Выбрать роль "Партнёр"
   - Не заполнять поле города
   - Проверить ошибку валидации

3. **Регистрация других ролей**:
   - Выбрать роль "Администратор" или "Эксперт"
   - Проверить что поле города не отображается
   - Проверить успешную регистрацию без города

4. **Смена роли в форме**:
   - Выбрать "Партнёр" → поле города появляется
   - Выбрать "Администратор" → поле города исчезает
   - Снова выбрать "Партнёр" → поле города появляется

## Совместимость

- Поле `city` опциональное, поэтому не влияет на существующих пользователей
- Для существующих партнеров поле будет `null` до обновления профиля
- API обратно совместимо - старые запросы без поля `city` работают для не-партнеров

Функциональность полностью готова к использованию.
