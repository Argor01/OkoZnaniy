# Интеграция системы тикетов с чатами поддержки

## Обзор

Реализована продвинутая система тикетов с:
- **Рандомными 16-символьными номерами** (например: A7K9M2X5P1Q8R3T6)
- **Назначением нескольких сотрудников** на один тикет
- **Системой тегов** с поддержкой хештегов (#негатив, #срочно, #баг)
- **Автоматической интеграцией с чатами поддержки**

## Новые возможности

### 🎯 Уникальные номера тикетов
- Каждый тикет получает уникальный 16-символьный номер
- Номер генерируется автоматически при создании
- Состоит из заглавных букв и цифр (A-Z, 0-9)
- Отображается вместо обычного ID во всех интерфейсах

### 👥 Назначение сотрудников
- Можно назначить несколько сотрудников на один тикет
- Назначать можно только пользователей с ролью admin или director
- Назначенные сотрудники отображаются в карточке тикета
- API для управления назначениями

### 🏷️ Система тегов
- Поддержка хештегов (#негатив, #срочно, #баг)
- Автоматическое добавление # если не указан
- Цветовая индикация (красный для #негатив)
- Быстрое добавление популярных тегов
- Отображение в списке и детальном просмотре

## API Endpoints

### Управление назначениями
- `POST /admin-panel/support-requests/{id}/assign_users/` - Назначить пользователей на запрос поддержки
- `POST /admin-panel/claims/{id}/assign_users/` - Назначить пользователей на претензию

### Управление тегами
- `POST /admin-panel/support-requests/{id}/add_tag/` - Добавить тег
- `POST /admin-panel/support-requests/{id}/remove_tag/` - Удалить тег
- `POST /admin-panel/support-requests/{id}/update_tags/` - Обновить все теги
- Аналогичные endpoints для претензий (`/claims/`)

### Получение данных
- `GET /admin-panel/users/?role=admin` - Список админов для назначения

## Использование

### Создание тикета с тегами
```python
# Backend
ticket = SupportRequest.objects.create(
    user=user,
    subject="Проблема с оплатой",
    description="...",
    tags="#негатив, #срочно, #оплата"
)

# Назначение сотрудников
ticket.assigned_users.set([admin1.id, admin2.id])
```

### Frontend управление
```typescript
// Назначение пользователей
await assignUsers(ticketId, [userId1, userId2], 'support_request');

// Добавление тега
await addTag(ticketId, '#негатив', 'support_request');

// Удаление тега
await removeTag(ticketId, '#негатив', 'support_request');
```

## Популярные теги

## Как открыть тикет в отдельной странице

### 1. Из списка тикетов в админ панели

- Кликните на любой тикет в списке - он откроется в отдельной странице
- URL: `/admin/tickets/{id}` (например: `/admin/tickets/123`)

### 2. Из детального просмотра тикета

- В правом верхнем углу есть кнопка "Открыть отдельно"
- Кнопка откроет тикет в новой вкладке

### 3. Прямая ссылка

- Можно напрямую перейти по URL: `http://localhost:5173/admin/tickets/{id}`
- Ссылка работает из любого места в системе

### 4. Из связанного чата поддержки

- В деталях тикета есть ссылка на связанный чат поддержки
- Из чата можно создать тикет и перейти к нему

- Сохранение состояния при навигации

### 1. Автоматическое создание тикетов

При создании нового чата поддержки автоматически создается соответствующий тикет:
- Тикет получает ту же тему и приоритет, что и чат
- Первое сообщение из чата копируется в описание тикета
- Тикет помечается как `auto_created = True`

### 2. Синхронизация сообщений

Все текстовые сообщения из чата автоматически синхронизируются с тикетом:
- Сообщения от клиентов и админов копируются в тикет
- Сохраняется информация об отправителе и времени

### 3. Ручное создание тикетов

В интерфейсе чата поддержки есть кнопка "Создать тикет":
- Позволяет создать тикет из существующего чата
- Копирует все сообщения из чата в тикет
- Предотвращает создание дублирующих тикетов

## Сигналы Django

Настроены автоматические сигналы:

1. **При создании чата** (`post_save` на `SupportChat`):
   - Создается тикет
   - Копируется первое сообщение

2. **При отправке сообщения** (`post_save` на `SupportMessage`):
   - Сообщение синхронизируется с тикетом

## Преимущества новой системы

1. **Единое место управления** - все обращения в тикетах
2. **Полная история** - все сообщения сохраняются в тикете
3. **Лучшее отслеживание** - статусы, приоритеты, назначения
4. **Аналитика** - можно анализировать время решения, нагрузку
5. **Эскалация** - возможность передавать тикеты между админами

## Workflow пользователя

1. **Пользователь создает чат поддержки**
2. **Автоматически создается тикет**
3. **Переписка ведется в чате**
4. **Админ управляет через тикеты**
5. **При необходимости можно создать дополнительный тикет**

## Совместимость

- Существующие чаты продолжают работать
- Старые тикеты остаются без изменений  
- Новая функциональность не ломает существующий код
- Можно постепенно мигрировать на новую систему

## Настройка

Для активации автоматического создания тикетов убедитесь, что:

1. Миграции применены: `python manage.py migrate`
2. Сигналы подключены в `apps/chat/apps.py`
3. Frontend обновлен с новыми компонентами
