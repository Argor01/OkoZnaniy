# Скрипт автоматического развертывания OkoZnaniy

## Описание

Безопасный скрипт для автоматического развертывания обновлений проекта на продакшн сервере.

## Возможности

✅ **Автоматическая проверка обновлений** - проверяет наличие новых коммитов в репозитории  
✅ **Создание бэкапов** - автоматически создает бэкапы перед развертыванием  
✅ **Откат при ошибках** - автоматически откатывается к предыдущей версии при ошибках  
✅ **Проверка здоровья** - проверяет работоспособность сервисов после развертывания  
✅ **Применение миграций** - автоматически применяет миграции базы данных  
✅ **Бэкап базы данных** - создает бэкап БД перед применением миграций  
✅ **Логирование** - все действия логируются в файл  
✅ **Очистка** - удаляет неиспользуемые Docker образы  

## Установка

### 1. Скопировать скрипт на сервер

```bash
# Из локальной машины
scp deploy.sh root@45.12.239.226:/root/OkoZnaniy/

# Или на сервере
cd /root/OkoZnaniy
# Создать файл deploy.sh и скопировать содержимое
```

### 2. Сделать скрипт исполняемым

```bash
chmod +x /root/OkoZnaniy/deploy.sh
```

### 3. Создать директорию для логов (если не существует)

```bash
sudo mkdir -p /var/log
sudo touch /var/log/okoznaniy-deploy.log
sudo chmod 644 /var/log/okoznaniy-deploy.log
```

## Использование

### Ручной запуск

```bash
cd /root/OkoZnaniy
./deploy.sh
```

### Автоматический запуск по расписанию (cron)

Для автоматического развертывания каждый день в 3:00 ночи:

```bash
# Открыть crontab
crontab -e

# Добавить строку (развертывание каждый день в 3:00)
0 3 * * * /root/OkoZnaniy/deploy.sh >> /var/log/okoznaniy-deploy.log 2>&1

# Или каждые 6 часов
0 */6 * * * /root/OkoZnaniy/deploy.sh >> /var/log/okoznaniy-deploy.log 2>&1

# Или каждый час
0 * * * * /root/OkoZnaniy/deploy.sh >> /var/log/okoznaniy-deploy.log 2>&1
```

### Webhook для автоматического развертывания при push в GitHub

Можно настроить webhook, который будет запускать скрипт при каждом push в репозиторий.

## Конфигурация

Основные параметры в начале скрипта:

```bash
PROJECT_DIR="/root/OkoZnaniy"           # Путь к проекту
BACKUP_DIR="/root/backups"              # Директория для бэкапов
LOG_FILE="/var/log/okoznaniy-deploy.log" # Файл логов
MAX_BACKUPS=5                           # Количество хранимых бэкапов
HEALTH_CHECK_TIMEOUT=60                 # Таймаут проверки здоровья (сек)
ROLLBACK_ON_ERROR=true                  # Откат при ошибках
```

## Что делает скрипт

1. **Проверка обновлений** - проверяет наличие новых коммитов в `origin/main`
2. **Создание бэкапа** - сохраняет текущий коммит и состояние контейнеров
3. **Подтягивание изменений** - выполняет `git pull origin main`
4. **Применение миграций** - проверяет и применяет миграции Django
5. **Бэкап базы данных** - создает дамп БД перед миграциями
6. **Сборка frontend** - пересобирает frontend контейнер
7. **Перезапуск сервисов** - перезапускает все контейнеры
8. **Проверка здоровья** - проверяет работоспособность backend и frontend
9. **Откат при ошибках** - откатывается к предыдущей версии если что-то пошло не так
10. **Очистка** - удаляет неиспользуемые Docker образы

## Безопасность

### Автоматический откат

Если на любом этапе происходит ошибка, скрипт автоматически:
- Откатывает git к предыдущему коммиту
- Пересобирает контейнеры
- Перезапускает сервисы

### Бэкапы

- Хранятся последние 5 бэкапов
- Каждый бэкап содержит информацию о коммите
- Бэкапы базы данных создаются перед миграциями

### Проверки

- Проверка существования директорий
- Проверка работоспособности сервисов
- Проверка успешности каждой операции
- Таймаут для проверки здоровья

## Логи

Все действия логируются в файл `/var/log/okoznaniy-deploy.log`

### Просмотр логов

```bash
# Последние 50 строк
tail -n 50 /var/log/okoznaniy-deploy.log

# Просмотр в реальном времени
tail -f /var/log/okoznaniy-deploy.log

# Поиск ошибок
grep "✗" /var/log/okoznaniy-deploy.log

# Поиск успешных развертываний
grep "Развертывание завершено успешно" /var/log/okoznaniy-deploy.log
```

## Ручной откат

Если нужно откатиться к предыдущей версии вручную:

```bash
cd /root/OkoZnaniy

# Найти последний бэкап
ls -lt /root/backups/okoznaniy_backup_*.commit | head -n 1

# Прочитать коммит из бэкапа
BACKUP_COMMIT=$(cat /root/backups/okoznaniy_backup_YYYYMMDD_HHMMSS.commit)

# Откатиться
git reset --hard $BACKUP_COMMIT
docker-compose build --no-cache
docker-compose up -d
```

## Мониторинг

### Проверка статуса последнего развертывания

```bash
# Последние 20 строк лога
tail -n 20 /var/log/okoznaniy-deploy.log

# Проверка наличия ошибок
if grep -q "✗" /var/log/okoznaniy-deploy.log | tail -n 100; then
    echo "Есть ошибки в последних развертываниях"
else
    echo "Ошибок нет"
fi
```

### Уведомления

Можно настроить отправку уведомлений в Telegram при ошибках:

```bash
# Добавить в конец скрипта deploy.sh перед main
send_telegram_notification() {
    local message="$1"
    local bot_token="YOUR_BOT_TOKEN"
    local chat_id="YOUR_CHAT_ID"
    
    curl -s -X POST "https://api.telegram.org/bot${bot_token}/sendMessage" \
        -d chat_id="${chat_id}" \
        -d text="${message}" \
        -d parse_mode="HTML" > /dev/null
}

# Использовать в функции rollback
send_telegram_notification "⚠️ <b>Ошибка развертывания OkoZnaniy</b>%0AВыполнен откат к предыдущей версии"
```

## Troubleshooting

### Скрипт не запускается

```bash
# Проверить права
ls -l deploy.sh
# Должно быть: -rwxr-xr-x

# Сделать исполняемым
chmod +x deploy.sh
```

### Ошибка "Permission denied"

```bash
# Запустить от root
sudo ./deploy.sh

# Или добавить в sudoers
sudo visudo
# Добавить: username ALL=(ALL) NOPASSWD: /root/OkoZnaniy/deploy.sh
```

### Контейнеры не запускаются

```bash
# Проверить логи
docker-compose logs

# Проверить статус
docker-compose ps

# Перезапустить вручную
docker-compose down
docker-compose up -d
```
