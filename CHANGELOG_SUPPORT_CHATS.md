# Изменения: Разделение чатов поддержки и обычных чатов

## Дата: 2 марта 2026

## Описание

Реализовано разделение чатов с технической поддержкой и обычных чатов между пользователями. Теперь чаты с поддержкой отображаются только в разделе "Чаты поддержки" в админ-панели и не видны на странице обычных сообщений.

## Изменения в коде

### Backend

#### `apps/chat/views.py`

1. **ChatViewSet.get_queryset()** - добавлена фильтрация чатов поддержки:
   - Исключаются чаты с пользователем `SUPPORT_USER_ID`
   - Исключаются чаты с маркерами "поддержка", "support", "техподдержка" в `context_title`

2. **Добавлена документация** к классам `ChatViewSet` и `SupportChatViewSet`

#### `config/settings.py`

Добавлена новая настройка:
```python
SUPPORT_USER_ID = os.getenv('SUPPORT_USER_ID', None)
```

### Конфигурация

#### `.env.example`

Добавлена переменная окружения:
```env
SUPPORT_USER_ID=
```

### Документация

Созданы новые файлы документации:

1. **`apps/chat/README.md`** - техническая документация системы чатов
2. **`docs/SUPPORT_CHATS_RU.md`** - инструкция для администраторов и пользователей
3. **`docs/TESTING_SUPPORT_CHATS.md`** - инструкция по тестированию

## Как это работает

### Для пользователей

- Чаты с технической поддержкой создаются через кнопку "Техническая поддержка"
- Эти чаты НЕ отображаются на странице "Сообщения"
- Чаты с поддержкой доступны в отдельном разделе "Поддержка"

### Для администраторов

- Все обращения в поддержку видны в разделе "Чаты поддержки"
- Можно взять обращение в работу, ответить и закрыть
- Чаты поддержки НЕ смешиваются с обычными чатами

## API Endpoints

### Обычные чаты
- `GET /api/chat/chats/` - список обычных чатов (без чатов поддержки)

### Чаты поддержки
- `GET /api/chat/support/` - список чатов поддержки
- `POST /api/chat/support/` - создать обращение
- `GET /api/chat/support/{id}/messages/` - получить сообщения
- `POST /api/chat/support/{id}/send_message/` - отправить сообщение
- `POST /api/chat/support/{id}/take_chat/` - взять в работу (админы)
- `POST /api/chat/support/{id}/close_chat/` - закрыть чат

## Миграции

Миграции не требуются, так как изменения касаются только логики фильтрации.

## Настройка

### Опционально: Установка ID пользователя поддержки

Добавьте в `.env`:
```env
SUPPORT_USER_ID=123
```

Где `123` - ID пользователя-администратора поддержки.
