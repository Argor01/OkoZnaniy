## Что было реализовано

### 🎯 Backend (Django)

#### Новое приложение `apps/arbitration`
- ✅ **ArbitrationCase** - модель арбитражных дел
  - Уникальный номер дела (ARB-XXXXXXXX)
  - Истец и ответчик
  - Связь с заказами
  - Финансовые требования (тип и процент возврата)
  - Статусы: черновик → подано → на рассмотрении → в арбитраже → решение принято → закрыто
  - Приоритеты: низкий, средний, высокий, срочный

- ✅ **ArbitrationMessage** - сообщения в делах
  - Типы: от истца, от ответчика, от администратора, системное
  - Внутренние сообщения (видны только админам)
  - Вложения

- ✅ **ArbitrationActivity** - лента активности
  - Все события: создание, изменение статуса, назначение админа, сообщения, решения

#### API Endpoints
```
GET    /api/arbitration/cases/                    - Список дел
POST   /api/arbitration/cases/submit-claim/       - Подача претензии
GET    /api/arbitration/cases/my-cases/           - Мои дела
POST   /api/arbitration/cases/{id}/take-in-work/  - Взять в работу
POST   /api/arbitration/cases/{id}/send-message/  - Отправить сообщение
POST   /api/arbitration/cases/{id}/process-refund/ - Оформить возврат
POST   /api/arbitration/cases/{id}/make-decision/ - Принять решение
POST   /api/arbitration/cases/{id}/close-case/    - Закрыть дело
POST   /api/arbitration/cases/{id}/update-status/ - Обновить статус
GET    /api/arbitration/cases/{id}/activity-feed/ - Лента активности
GET    /api/arbitration/stats/                    - Статистика
```

## Структура данных

### Статусы дела
1. `draft` - Черновик
2. `submitted` - Подано
3. `under_review` - На рассмотрении
4. `awaiting_response` - Ожидает ответа
5. `in_arbitration` - В арбитраже
6. `decision_made` - Решение принято
7. `closed` - Закрыто
8. `rejected` - Отклонено

### Причины обращения
- Заказ не выполнен
- Низкое качество работы
- Нарушение сроков
- Спор по оплате
- Нарушение условий договора
- Другое

### Типы возврата
- Без возврата
- Частичный возврат (с указанием процента)
- Полный возврат
