# Landing Page Styles - Modular Structure

## 📁 Структура файлов

```
src/styles/landing/
├── index.css                 # Главный файл импортов
├── base/
│   ├── variables.css        # CSS переменные
│   ├── reset.css           # Базовые сбросы стилей
│   ├── typography.css      # Импорт шрифтов
│   └── utilities.css       # Утилитарные классы
├── components/
│   ├── buttons.css         # Стили кнопок
│   ├── header.css          # Навигация и шапка
│   └── footer.css          # Подвал сайта
└── sections/
    ├── first-screen.css    # Главный экран
    ├── place-task.css      # Размещение задач
    ├── advantages.css      # Преимущества
    ├── prices.css          # Цены
    ├── only-pro.css        # Только профессионалы
    ├── reviews.css         # Отзывы
    ├── leave-order.css     # Оставить заказ
    ├── faq.css             # FAQ
    └── place-task-info.css # Информация о размещении
```

## 🎨 CSS Переменные

### Цвета
- `--color-white` - белый цвет
- `--color-black` - черный цвет
- `--color-grey-900` - темно-серый
- `--color-grey-700` - серый
- `--color-grey-600` - средне-серый
- `--color-brand-blue-*` - оттенки синего бренда
- `--color-brand-orange-*` - оттенки оранжевого бренда
- `--color-brand-purple-*` - оттенки фиолетового бренда

### Типографика
- `--font-family-sans` - основной шрифт (Jost)
- `--font-family-display` - заголовочный шрифт (Alegreya)

### Размеры
- `--container-max-width` - максимальная ширина контейнера
- `--container-padding` - отступы контейнера
- `--radius-*` - радиусы скругления

### Breakpoints
- `--breakpoint-mobile` - 576px
- `--breakpoint-tablet` - 768px
- `--breakpoint-desktop` - 1024px
- `--breakpoint-large` - 1320px

## 🛠 Утилитарные классы

- `.mcontainer` - основной контейнер
- `.main` - основной контент
- `.card-shadow` - тень для карточек
- `.flex-center` - центрирование по flex
- `.text-white` - белый текст
- `.bg-white` - белый фон
- `.border-radius-10` - скругление 10px

## 📦 Использование

### В React компонентах:
```tsx
// Импорт всех стилей лендинга
import '../styles/landing/index.css';

// Или импорт конкретного модуля
import '../styles/landing/sections/first-screen.css';
```

### В основном CSS файле:
```css
/* Импорт всей модульной структуры */
@import './landing/index.css';
```

## ✅ Преимущества модульной структуры

1. **Читаемость** - каждый файл отвечает за свою область
2. **Поддержка** - легко найти и изменить нужные стили
3. **Переиспользование** - компоненты можно использовать отдельно
4. **Командная работа** - меньше конфликтов при слиянии
5. **Производительность** - возможность ленивой загрузки
6. **Масштабируемость** - легко добавлять новые модули

## 🔧 Рекомендации по разработке

1. **Используйте CSS переменные** вместо хардкод значений
2. **Применяйте утилитарные классы** для часто используемых стилей
3. **Следуйте BEM методологии** для именования классов
4. **Группируйте медиа-запросы** внутри селекторов
5. **Документируйте сложные стили** комментариями

## 📝 Миграция с монолитного файла

Старый файл `landing.css` теперь импортирует модульную структуру:
```css
/* Landing Page Styles - Import Modular Structure */
@import './landing/index.css';
```

Это обеспечивает обратную совместимость и плавную миграцию.
