# Система арбитража

Профессиональная система арбитража для разрешения споров между пользователями платформы.

## Основные возможности

### Для клиентов

1. **Пошаговая подача претензии**
   - Шаг 1: Основная информация (заказ, ответчик, тема)
   - Шаг 2: Причина и подробное описание проблемы
   - Шаг 3: Финансовые требования (тип и процент возврата)

2. **Обязательные поля**
   - Описание проблемы (минимум 50 символов)
   - Причина обращения
   - Тип возврата

3. **Отслеживание статуса**
   - Черновик → Подано → На рассмотрении → В арбитраже → Решение принято → Закрыто

### Для администраторов

1. **Структурированный просмотр**
   - Четкое разделение: истец, ответчик, дата подачи
   - Вся история переписки в одном окне
   - Параметры претензии в столбец (не "кашей")

2. **Работающие кнопки**
   - "Взять в работу" - меняет статус и назначает администратора
   - "Оформить возврат" - с указанием процента возврата
   - "Принять решение" - с текстом решения и одобренным процентом
   - "Закрыть дело" - с финальным сообщением

3. **Исправленные баги**
   - Поле ввода текста работает корректно (не по одной букве)
   - Кнопки быстрых действий меняют статус
   - Отправка сообщений без ошибок

## API Endpoints

### Для клиентов

```
POST /api/arbitration/cases/submit-claim/
```
Подача претензии (пошаговая форма)

```json
{
  "order_id": 123,
  "defendant_id": 456,
  "subject": "Заказ не выполнен в срок",
  "reason": "deadline_violation",
  "description": "Подробное описание проблемы...",
  "deadline_relevant": true,
  "refund_type": "partial",
  "requested_refund_percentage": 50,
  "evidence_files": []
}
```

```
GET /api/arbitration/cases/my-cases/
```
Получить все дела текущего пользователя

### Для администраторов

```
GET /api/arbitration/cases/
```
Список всех дел (с фильтрами)

Параметры:
- `status` - фильтр по статусу
- `priority` - фильтр по приоритету
- `assigned_to_me=true` - только назначенные на меня

```
GET /api/arbitration/cases/{id}/
```
Детальная информация о деле

```
POST /api/arbitration/cases/{id}/take-in-work/
```
Взять дело в работу

```
POST /api/arbitration/cases/{id}/send-message/
```
Отправить сообщение

```json
{
  "message": "Текст сообщения",
  "is_internal": false
}
```

```
POST /api/arbitration/cases/{id}/process-refund/
```
Оформить возврат

```json
{
  "refund_percentage": 50,
  "refund_amount": 5000
}
```

```
POST /api/arbitration/cases/{id}/make-decision/
```
Принять решение

```json
{
  "decision": "Текст решения",
  "approved_refund_percentage": 50,
  "approved_refund_amount": 5000
}
```

```
POST /api/arbitration/cases/{id}/close-case/
```
Закрыть дело

```json
{
  "message": "Финальное сообщение клиенту"
}
```

```
POST /api/arbitration/cases/{id}/update-status/
```
Обновить статус

```json
{
  "status": "in_arbitration"
}
```

```
GET /api/arbitration/cases/{id}/activity-feed/
```
Получить объединенную ленту сообщений и активностей

```
GET /api/arbitration/stats/
```
Статистика по арбитражу

## Модели данных

### ArbitrationCase
Основная модель арбитражного дела

Поля:
- `case_number` - уникальный номер дела (ARB-XXXXXXXX)
- `plaintiff` - истец
- `defendant` - ответчик
- `order` - связанный заказ
- `reason` - причина обращения
- `subject` - тема
- `description` - описание проблемы
- `status` - статус дела
- `priority` - приоритет
- `refund_type` - тип возврата
- `requested_refund_percentage` - запрошенный процент
- `approved_refund_percentage` - одобренный процент
- `decision` - текст решения
- `assigned_admin` - назначенный администратор

### ArbitrationMessage
Сообщения в деле

Типы сообщений:
- `plaintiff` - от истца
- `defendant` - от ответчика
- `admin` - от администратора
- `system` - системное

### ArbitrationActivity
Лента активности дела

Типы активности:
- `created` - дело создано
- `submitted` - дело подано
- `status_changed` - статус изменен
- `admin_assigned` - назначен администратор
- `message_sent` - отправлено сообщение
- `decision_made` - принято решение
- `refund_processed` - оформлен возврат
- `closed` - дело закрыто

## Отличия от техподдержки

1. **Отдельная сущность** - арбитраж полностью отделен от техподдержки
2. **Две стороны** - истец и ответчик (не клиент-админ)
3. **Финансовые требования** - встроенная система возвратов
4. **Пошаговая подача** - структурированный процесс создания претензии
5. **Расширенная история** - объединенная лента сообщений и активностей
6. **Профессиональный UI** - современный дизайн как в крупных проектах

## Безопасность

- Клиенты видят только свои дела (как истец или ответчик)
- Администраторы видят все дела
- Внутренние сообщения видны только администраторам
- Все действия логируются в ленту активности

## Уведомления

Рекомендуется настроить уведомления для:
- Подачи новой претензии
- Получения сообщения от администратора
- Принятия решения по делу
- Закрытия дела

## Интеграция с существующей системой

Арбитраж интегрируется с:
- Системой заказов (Order)
- Системой пользователей (User)
- Системой уведомлений (Notifications)
- Админ-панелью (AdminPanel)
