# Партнерские реферальные ссылки

## Описание функциональности

Каждый партнер получает уникальную реферальную ссылку с реферальным кодом. При переходе по этой ссылке пользователь попадает на страницу регистрации с автоматически заполненным реферальным кодом.

## Как это работает

### 1. Генерация реферального кода

При создании партнера (пользователя с ролью `partner`) автоматически генерируется уникальный реферальный код:

```python
# apps/users/models.py
def save(self, *args, **kwargs):
    # Генерируем реферальный код для партнеров
    if self.role == 'partner' and not self.referral_code:
        import uuid
        self.referral_code = str(uuid.uuid4())[:8].upper()
    super().save(*args, **kwargs)
```

### 2. Отображение партнерской ссылки

В личном кабинете партнера (раздел "Промоматериалы") отображается:
- Реферальный код
- Партнерская ссылка вида: `https://okoznaniy.ru/login?ref=ABC12345`
- Кнопки для копирования кода и ссылки
- Статистика по рефералам

### 3. Переход по реферальной ссылке

Поддерживаются следующие форматы ссылок:
- `/login?ref=CODE` - прямая ссылка на страницу входа/регистрации
- `/ref/CODE` - короткая ссылка (редирект на /login)
- `/referral/CODE` - альтернативная ссылка (редирект на /login)
- `/r/CODE` - самая короткая ссылка (редирект на /login)

При переходе по любой из этих ссылок:
1. Реферальный код сохраняется в `localStorage`
2. Пользователь перенаправляется на страницу входа/регистрации (`/login`)
3. Реферальный код автоматически подставляется в форму регистрации

### 4. Регистрация с реферальным кодом

На странице регистрации:
- Если в URL есть параметр `ref`, он автоматически подставляется в форму
- Отображается уведомление о том, что пользователь регистрируется по приглашению партнера
- При отправке формы реферальный код передается на бэкенд

### 5. Обработка на бэкенде

При регистрации с реферальным кодом:

```python
# apps/users/serializers.py
def create(self, validated_data):
    referral_code = validated_data.pop('referral_code', None)
    # ...
    
    # Если указан реферальный код, находим партнера
    if referral_code:
        try:
            partner = User.objects.get(
                referral_code=referral_code,
                role='partner'
            )
            user.partner = partner
            
            # Увеличиваем счетчик рефералов у партнера
            partner.total_referrals += 1
            partner.save(update_fields=['total_referrals'])
            
        except User.DoesNotExist:
            # Если код не найден, просто игнорируем
            pass
```

## API Endpoints

### Получение партнерской ссылки

```
POST /api/users/generate_referral_link/
Authorization: Bearer <token>
```

Ответ:
```json
{
  "referral_code": "ABC12345",
  "referral_link": "https://okoznaniy.ru/login?ref=ABC12345"
}
```

### Регистрация с реферальным кодом

```
POST /api/users/
Content-Type: application/json

{
  "username": "newuser",
  "email": "user@example.com",
  "password": "password123",
  "role": "client",
  "referral_code": "ABC12345"
}
```

## Компоненты

### Frontend

1. **PromoMaterials.tsx** - компонент для отображения промоматериалов партнера
   - Вкладка "Партнерская ссылка" с кодом и ссылкой
   - Кнопки копирования
   - Статистика по рефералам

2. **ReferralRedirect.tsx** - компонент для обработки коротких реферальных ссылок
   - Сохраняет код в localStorage
   - Перенаправляет на страницу регистрации

3. **RegisterWithEmailVerification.tsx** - страница регистрации
   - Автоматически подставляет реферальный код из URL
   - Отображает уведомление о регистрации по приглашению

### Backend

1. **User model** - модель пользователя с полями:
   - `referral_code` - уникальный реферальный код партнера
   - `partner` - ссылка на партнера (реферера)
   - `total_referrals` - общее количество рефералов
   - `active_referrals` - количество активных рефералов
   - `partner_commission_rate` - процент комиссии партнера

2. **CustomRegisterSerializer** - сериализатор регистрации
   - Обрабатывает поле `referral_code`
   - Связывает нового пользователя с партнером
   - Обновляет статистику партнера

## Тестирование

### Создание тестового партнера

```bash
python manage.py shell
```

```python
from apps.users.models import User

# Создаем партнера
partner = User.objects.create_user(
    username='testpartner',
    email='partner@test.com',
    password='testpass123',
    role='partner'

print(f"Реферальный код: {partner.referral_code}")
print(f"Партнерская ссылка: http://localhost:3000/login?ref={partner.referral_code}")
```

### Проверка регистрации по реферальной ссылке

1. Откройте ссылку: `http://localhost:3000/login?ref=ABC12345`
2. Убедитесь, что реферальный код отображается на странице (на вкладке регистрации)
3. Зарегистрируйте нового пользователя
4. Проверьте, что у нового пользователя установлено поле `partner`
5. Проверьте, что у партнера увеличился счетчик `total_referrals`

```python
# Проверка в shell
from apps.users.models import User

new_user = User.objects.get(username='newuser')
print(f"Партнер: {new_user.partner}")
print(f"Реферальный код партнера: {new_user.partner.referral_code}")

partner = User.objects.get(username='testpartner')
print(f"Всего рефералов: {partner.total_referrals}")
```

## Безопасность

- Реферальный код генерируется автоматически и является уникальным
- Если реферальный код не найден, регистрация продолжается без привязки к партнеру
- Реферальный код не является секретным и может быть публичным
- Партнер не может изменить свой реферальный код после создания
