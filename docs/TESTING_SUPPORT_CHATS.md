# Инструкция по тестированию чатов поддержки

## Что было изменено

Теперь чаты с технической поддержкой **НЕ отображаются** на странице обычных чатов пользователей. Они видны только в разделе **"Чаты поддержки"** в админ-панели.

## Как протестировать

### 1. Настройка (опционально)

Если хотите исключить чаты с конкретным пользователем поддержки, добавьте в `.env`:

```env
SUPPORT_USER_ID=123
```

Где `123` - это ID пользователя-администратора поддержки.

### 2. Тестирование от имени пользователя

#### Создание обращения в поддержку

1. Войдите на сайт как обычный пользователь (клиент или эксперт)
2. Найдите кнопку **"Техническая поддержка"** (обычно в правом нижнем углу)
3. Нажмите на кнопку
4. Заполните форму:
   - Выберите тему обращения
   - Укажите приоритет
   - Опишите проблему
5. Нажмите **"Отправить"**
6. Вы будете перенаправлены в чат с поддержкой

#### Проверка списка чатов

1. Перейдите на страницу **"Сообщения"** или **"Чаты"**
2. **Убедитесь, что чат с поддержкой НЕ отображается в этом списке**
3. Здесь должны быть только обычные чаты с экспертами/клиентами

#### Просмотр чатов поддержки

1. Перейдите в раздел **"Поддержка"** или **"Support"** (если есть отдельная страница)
2. Здесь должны отображаться ваши обращения в техническую поддержку
3. Откройте чат и отправьте сообщение

### 3. Тестирование от имени администратора

#### Просмотр обращений

1. Войдите в админ-панель
2. Перейдите в раздел **"Чаты поддержки"** или **"Support Chats"**
3. Вы должны видеть все обращения пользователей
4. Проверьте информацию:
   - Имя клиента
   - Тема обращения
   - Статус
   - Приоритет
   - Последнее сообщение

#### Работа с обращением

1. Откройте любое обращение
2. Нажмите **"Взять в работу"** (если доступно)
3. Отправьте ответ пользователю
4. Проверьте, что сообщение отправлено
5. Закройте обращение кнопкой **"Закрыть чат"**

#### Проверка фильтрации

1. Перейдите на страницу обычных чатов в админ-панели (если есть)
2. **Убедитесь, что чаты поддержки НЕ отображаются в списке обычных чатов**

### 4. Проверка API

#### Получение списка обычных чатов

```bash
# Должны вернуться только обычные чаты (без чатов поддержки)
curl -H "Authorization: Bearer YOUR_TOKEN" http://localhost:8000/api/chat/chats/
```

#### Получение списка чатов поддержки

```bash
# Для обычного пользователя - только свои чаты
curl -H "Authorization: Bearer YOUR_TOKEN" http://localhost:8000/api/chat/support/

# Для админа - все чаты поддержки
curl -H "Authorization: Bearer ADMIN_TOKEN" http://localhost:8000/api/chat/support/
```

### 5. Проверка в базе данных

```bash
# Войдите в контейнер
docker-compose exec backend python manage.py shell

# Проверьте количество чатов
>>> from apps.chat.models import Chat, SupportChat
>>> from apps.users.models import User

# Получите пользователя
>>> user = User.objects.get(email='test@example.com')

# Обычные чаты (должны быть без чатов поддержки)
>>> Chat.objects.filter(participants=user).count()

# Чаты поддержки
>>> SupportChat.objects.filter(client=user).count()

# Проверьте, что чаты с SUPPORT_USER_ID исключены
>>> from django.conf import settings
>>> if settings.SUPPORT_USER_ID:
...     Chat.objects.filter(participants=user, participants__id=settings.SUPPORT_USER_ID).count()
...     # Должно быть 0
```

## Ожидаемые результаты

### ✅ Правильное поведение

1. Чаты с технической поддержкой НЕ отображаются на странице "Сообщения"
2. Чаты с технической поддержкой отображаются в разделе "Чаты поддержки" в админ-панели
3. Обычные пользователи видят только свои чаты с поддержкой
4. Администраторы видят все чаты с поддержкой
5. API `/api/chat/chats/` возвращает только обычные чаты
6. API `/api/chat/support/` возвращает только чаты поддержки

### ❌ Неправильное поведение (нужно исправить)

1. Чаты с поддержкой отображаются на странице "Сообщения"
2. Чаты с поддержкой не отображаются в админ-панели
3. Обычные пользователи видят чужие чаты с поддержкой
4. API возвращает смешанные результаты

## Возможные проблемы

### Чаты поддержки все еще отображаются в обычных чатах

**Решение:**
1. Проверьте, что контейнеры пересобраны: `docker-compose down && docker-compose build --no-cache backend && docker-compose up -d`
2. Проверьте настройку `SUPPORT_USER_ID` в `.env`
3. Проверьте, что чаты не имеют маркеров "поддержка", "support", "техподдержка" в `context_title`

### Не могу создать обращение в поддержку

**Решение:**
1. Проверьте, что кнопка "Техническая поддержка" отображается на фронтенде
2. Проверьте логи бэкенда: `docker-compose logs backend --tail=50`
3. Проверьте, что API `/api/chat/support/` доступен

### Администратор не видит обращения

**Решение:**
1. Проверьте роль пользователя: должна быть `role='admin'`
2. Проверьте, что обращения созданы через `SupportChat`, а не через `Chat`
3. Проверьте API: `curl -H "Authorization: Bearer ADMIN_TOKEN" http://localhost:8000/api/chat/support/`

## Контакты

Если возникли проблемы при тестировании, проверьте:
- Логи бэкенда: `docker-compose logs backend`
- Логи фронтенда: `docker-compose logs frontend`
- Консоль браузера (F12)
