# Диагностика проблем с доставкой email

## Проблема
Некоторые пользователи не получают письма с кодом подтверждения при регистрации.

## Текущее состояние
- Email отправка работает корректно (SMTP Яндекс подключается успешно)
- Коды создаются и сохраняются в базе
- Логи показывают успешную отправку писем

## Возможные причины

### 1. Письма попадают в папку "Спам"
**Решение**: Попросите пользователей проверить папку "Спам"

### 2. Блокировка писем почтовыми провайдерами
Некоторые почтовые провайдеры могут блокировать письма:
- **gmail.com** - строгая спам-фильтрация
- **mail.ru** - может блокировать письма с Яндекса
- **корпоративные почты** - могут блокировать внешние рассылки

**Решение**: 
- Добавить SPF, DKIM, DMARC записи для домена okoznaniy.ru
- Использовать специализированный email сервис (SendGrid, Mailgun, Amazon SES)

### 3. Пользователи вводят неправильный email
**Решение**: 
- Добавить подтверждение email через повторный ввод
- Показать первые и последние символы email для проверки

### 4. Лимиты Яндекс.Почты
Яндекс имеет лимиты на отправку писем:
- До 1000 писем в день для бесплатных аккаунтов
- При превышении лимита письма могут задерживаться

## Обновления

### 2026-03-28 - Найдена критическая проблема!

**Проблема**: SPF запись домена не соответствует используемому SMTP серверу

**Текущие DNS записи**:
- SPF: `v=spf1 redirect=beget.com` (указывает на Beget)
- MX: `mx1.beget.com`, `mx2.beget.com` (Beget)
- SMTP для отправки: `smtp.yandex.ru` (Яндекс)

**Решение**:
1. Обновить SPF запись домена okoznaniy.ru:
   ```
   v=spf1 include:_spf.yandex.ru ~all
   ```
   или если нужно оставить и Beget:
   ```
   v=spf1 include:_spf.beget.com include:_spf.yandex.ru ~all
   ```

2. Для улучшения доставки на gmail.com рекомендуется:
   - Настроить DKIM подпись в Яндекс.Почте для домена
   - Добавить DMARC запись:
     ```
     _dmarc.okoznaniy.ru IN TXT "v=DMARC1; p=quarantine; rua=mailto:admin@okoznaniy.ru"
     ```
