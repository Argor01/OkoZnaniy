# Изменения: Единый дашборд для всех пользователей

## Что изменено

### ❌ Удалено
- **ClientDashboard** больше не используется
- Отдельный дашборд для клиентов

### ✅ Добавлено
- Все пользователи (клиенты и эксперты) теперь используют **ExpertDashboard**
- Перенаправление `/dashboard` → `/expert`

## Изменения в файлах

### 1. `frontend-react/src/pages/Login.tsx`
Все перенаправления после авторизации ведут на `/expert`:

```typescript
// Было:
if (role === 'client') {
  await redirectClient(); // → /dashboard или /create-order
} else if (role === 'expert') {
  navigate('/expert');
}

// Стало:
if (role === 'client' || role === 'expert') {
  navigate('/expert');
}
```

### 2. `frontend-react/src/App.tsx`
- Удален импорт `ClientDashboard`
- Маршрут `/dashboard` теперь перенаправляет на `/expert`:

```typescript
<Route 
  path="/dashboard" 
  element={<Navigate to="/expert" replace />} 
/>
```

### 3. `frontend-react/src/components/layout/DashboardLayout.tsx`
- Убран пункт меню "Мои заказы" для клиентов
- Клиенты видят только "Создать заказ" в упрощенном сайдбаре

## Поведение после изменений

### При авторизации клиента:
1. Вход через `/login`
2. Перенаправление на `/expert`
3. Открывается ExpertDashboard со своим расширенным сайдбаром
4. Клиент видит все функции, как и эксперт

### При авторизации эксперта:
1. Вход через `/login`
2. Перенаправление на `/expert`
3. Открывается ExpertDashboard со своим расширенным сайдбаром
4. Эксперт видит все функции

### Навигация:
- `/expert` - главный дашборд (ExpertDashboard)
- `/create-order` - создание заказа (с DashboardLayout)
- `/dashboard` - автоматически перенаправляет на `/expert`

2. Войдите как эксперт → должны попасть на `/expert`

4. Проверьте, что все функции работают для обеих ролей
