## 1. Обзор проекта

VK-бот для платформы OkoZnaniy — сервис для отправки уведомлений пользователям через сообщения сообщества ВКонтакте. Бот интегрируется с существующей системой уведомлений Django и обеспечивает push-доставку важных событий: системные уведомления с сайта, новые сообщения в чатах, обновления заказов и другие события.

### Ключевые возможности
- **Push-уведомления** — мгновенная доставка системных уведомлений в ВК
- **Уведомления о сообщениях** — оповещение о новых сообщениях в чате
- **Авторизация** — вход на сайт через VK-бота (deep link)
- **Профиль и баланс** — просмотр данных аккаунта из VK
- **Настройка уведомлений** — управление подпиской на различные типы уведомлений

### Логика отправки

```python
# Псевдокод
def on_notification_created(notification):
    user = notification.recipient
    
    # 1. Проверяем, есть ли vk_id
    if not user.vk_id:
        return
    
    # 2. Проверяем, включены ли VK-уведомления
    if not user.vk_notifications_enabled:
        return
    
    # 3. Форматируем сообщение
    text = format_vk_notification(notification)
    
    # 4. Отправляем через Celery (асинхронно)
    send_vk_notification.delay(user.vk_id, text)
```

## 8. Интеграция с чатом

При получении нового сообщения в чате, если получатель не онлайн на сайте и имеет привязанный `vk_id`, отправляется push-уведомление:

```
💬 Новое сообщение

От: Иван Петров
Чат: Заказ №42 — Курсовая работа по экономике

"Здравствуйте, уточните пожалуйста сроки..."

[Открыть чат]
```

### Защита от спама
- Не отправляем push, если пользователь онлайн на сайте (проверка через WebSocket)
- Rate limiting: не чаще 1 push в 30 секунд на один чат
- Группировка: если пришло несколько сообщений подряд, отправляем одно уведомление

## 9. Переменные окружения

| Переменная | Описание | Обязательная | Пример |
|-----------|----------|-------------|--------|
| `VK_BOT_TOKEN` | Токен сообщества VK | Да | `vk1.a.xxx...` |
| `VK_GROUP_ID` | ID группы VK | Да | `123456789` |
| `VK_SERVICE_KEY` | Сервисный ключ VK (уже есть) | Нет | `xxx...` |
| `VK_API_VERSION` | Версия VK API | Нет (по умолч. 5.199) | `5.199` |
| `DATABASE_URL` | Строка подключения к PostgreSQL | Да | `postgresql://...` |
| `REDIS_URL` | Строка подключения к Redis | Да | `redis://...` |
| `SECRET_KEY` | Django secret key | Да | - |
| `FRONTEND_URL` | URL фронтенда | Да | `https://okoznaniy.ru` |

### Получение токена VK

1. Создать сообщество ВКонтакте (или использовать существующее)
2. Перейти в **Управление → Работа с API**
3. Создать ключ доступа с правами:
   - Доступ к сообщениям сообщества
   - Доступ к управлению сообществом
4. Включить **Long Poll API** в настройках сообщества
5. В настройках Long Poll API включить событие `message_new`
6. Включить возможность **Сообщения сообщества** в настройках

## 10. Docker-инфраструктура

### Dockerfile.vkbot

```dockerfile
FROM python:3.11-slim

WORKDIR /app

RUN apt-get update && apt-get install -y \
    gcc \
    postgresql-client \
    && rm -rf /var/lib/apt/lists/*

COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt

COPY . .

CMD ["python", "vk_bot/bot.py"]
```

### docker-compose.yml — новый сервис

```yaml
vk-bot:
  build:
    context: .
    dockerfile: Dockerfile.vkbot
  restart: unless-stopped
  depends_on:
    backend:
      condition: service_started
    postgres:
      condition: service_healthy
    redis:
      condition: service_started
  environment:
    DATABASE_URL: postgresql://postgres:${POSTGRES_PASSWORD}@postgres:5432/oko_db
    REDIS_URL: redis://:${REDIS_PASSWORD}@redis:6379/0
    SECRET_KEY: ${SECRET_KEY}
    VK_BOT_TOKEN: ${VK_BOT_TOKEN}
    VK_GROUP_ID: ${VK_GROUP_ID}
    VK_SERVICE_KEY: ${VK_SERVICE_KEY}
    FRONTEND_URL: ${FRONTEND_URL:-http://localhost:5173}
    DEBUG: ${DEBUG:-False}
    PYTHONPATH: /app
    DJANGO_SETTINGS_MODULE: config.settings
  networks:
    - app_network
```

## 12. Безопасность

- **Токены** хранятся только в переменных окружения, никогда в коде
- **VK ID** валидируется через VK API при привязке
- **Rate limiting** на отправку сообщений (VK лимит: 20 сообщений/сек)
- **Авторизация** через одноразовые коды (`auth_id`) с TTL 5 минут в Redis
- **Логирование** без конфиденциальных данных
- **Celery retry** с экспоненциальной задержкой при ошибках VK API
