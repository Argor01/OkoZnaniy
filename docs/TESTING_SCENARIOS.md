# Сценарии ручного тестирования

## Подготовка
- Создать учетные записи: Client A, Expert A, Expert B.
- У Client A должна быть возможность создать заказ.
- Убедиться, что заказы создаются со статусом `new`, без назначенного эксперта.

## Базовые данные для тестов
- Заказ 1: описание заполнено, есть предмет, тип работы, дедлайн, бюджет, минимум 1 файл.
- Заказ 2: описание заполнено, без файлов.

## Лента заказов (эксперт)
- Открыть ленту от имени Expert A → карточки отображаются.
- Карточка содержит описание заказа.
- Отображаются предмет, тип работы, тема (если есть).
- Отображается бюджет и сроки.
- Отображается информация о заказчике (имя/аватар).
- Отображается счетчик откликов.
- При наличии файлов — видны файлы и доступна загрузка.
- Кнопка «Откликнуться» активна для заказов без отклика эксперта.
- После отклика кнопка отображается как «Вы уже откликнулись».

## Проваливание в заказ из ленты (эксперт)
- Клик по карточке заказа открывает страницу заказа.
- Заказ не помечается как удаленный/недоступный.
- На странице заказа видны описание, файлы, данные клиента.
- Файлы можно скачать.

## Доступ к заказу (эксперт)
- Эксперт может открыть заказ со статусом `new` без назначенного эксперта.
- Эксперт не может открыть заказ, в котором он не участник, если статус не `new` или эксперт уже назначен.

## Личный кабинет клиента → Заказы
- В списке «Мои размещенные заказы» есть кнопка удаления на каждой карточке.
- Удаление подтверждается диалогом.
- После удаления заказ исчезает из списка.

## Удаление заказа (сквозной сценарий)
- Удалить заказ Client A.
- Проверить, что удаленный заказ исчез из ленты у Expert A и Expert B.
- Проверить, что удаленный заказ не отображается в «Доступных заказах».
- Открыть прямую ссылку на удаленный заказ → сообщение о недоступности и редирект.

## Файлы заказа
- Эксперт видит файлы у заказа в статусе `new` без назначенного эксперта.
- Эксперт может скачать файл из ленты и со страницы заказа.
- Клиент и назначенный эксперт всегда имеют доступ к файлам.

## Чаты: индивидуальные предложения
- Expert A отправляет offer Expert B → Expert B принимает → создается заказ.
- Client A принимает offer Expert A → создается заказ.
- Нельзя принять собственный offer.
- Отклонение offer доступно участнику, не являющемуся автором сообщения.

## Чаты: work_offer / delivery
- Участник, не являющийся автором сообщения, может принять/отклонить delivery.
- Автор сообщения не может принять свое delivery.

## Админские кабинеты
### Админ: Заказы
- Открыть список всех заказов → список загружается без ошибок.
- Открыть карточку заказа → отображаются данные заказа и клиента.
- Фильтрация по статусу работает.
- Сортировка по дате создания работает.

### Админ: Пользователи и роли
- Открыть список пользователей → отображаются роли и статусы.
- Поиск по пользователю работает.
- Изменение роли пользователя сохраняется.

### Админ: Проблемные заказы
- Открыть раздел проблемных заказов → список формируется.
- Просмотр деталей проблемного заказа работает.

### Админ: Уведомления/настройки
- Открыть раздел настроек уведомлений → данные загружаются.
- Изменение настроек сохраняется.

### Арбитраж
- Арбитр видит список споров.
- Открытие карточки спора работает.
- После изменения статуса спор исчезает из списка активных.

### Директор
- Открыть дашборд директора → основные показатели загружаются.
- Раздел статистики отображается без ошибок.
- Фильтрация/переключение периодов отражает изменения в метриках.

### Партнер
- Открыть кабинет партнера → данные и разделы загружаются.
- Отображается список закрепленных/доступных объектов (если предусмотрено).
- Фильтрация/поиск в кабинете партнера работает.

## Негативные сценарии
- Пользователь без роли expert не может открыть список доступных заказов.
- Эксперт не может открыть заказ в работе, если он не участник.
